Troubleshooting WSL

WSL is usually stable, but sometimes installation or runtime issues appear. This page covers common problems and how to solve them.

General diagnostics

When something is wrong, start by checking the overall WSL configuration: 

wsl --status
wsl --list --verbose

This shows which version (1 or 2) each distribution is using and whether system components are installed correctly.

WSL fails to start

If WSL does not start at all, try:

  1. Ensure that Virtualization is enabled in your BIOS/UEFI.

  3. Run wsl --shutdown and then try again.
  4. Reboot Windows.

Networking issues

If WSL cannot reach the internet:

  • Check if regular Windows apps have internet access.
  • Restart WSL: 
    wsl --shutdown
  • Try disabling and re-enabling your main network adapter.
  • If you use a VPN, temporarily disconnect it to see if it is blocking WSL traffic.

‚ÄúFile system is slow‚Äù problems

If operations on your codebase feel very slow:

  • Move the repository from /mnt/c/... to the Linux filesystem (e.g. /home/<user>/projects).
  • Avoid scanning Windows drives with tools that read many small files repeatedly (like certain linters or search tools).

Resetting a broken distribution

If a distribution is totally broken but you want to keep others, you can unregister it: 

wsl --unregister <DistributionName>

Warning: Unregistering deletes all data in that distribution. Back up your important files before doing this.

Exporting and importing for backup

You can back up an entire distribution by exporting it to a .tar file: 

# Export
wsl --export <DistributionName> C:\backups\my-distro.tar

# Import as a new distribution
wsl --import my-restored-distro C:\wsl\my-restored C:\backups\my-distro.tar --version 2

Logging and event viewer

For deeper debugging, check the Windows Event Viewer:

  1. Press Win + R, type eventvwr, press Enter.
  2. Under Windows Logs > System, look for entries related to WSL, Hyper-V, or Virtualization.
