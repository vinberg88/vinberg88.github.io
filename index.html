Windows Subsystem for Linux (WSL)

Windows Subsystem for Linux (WSL) lets you run a real Linux environment directly on Windows, without a virtual machine and without dual-booting. It is ideal for developers who want the power of Linux tools together with their existing Windows workflow.

What is WSL?

WSL is a compatibility layer in Windows that allows you to run Linux user-space binaries natively. Instead of emulating or virtualizing an entire Linux system, WSL integrates Linux with Windows so you can:

  • Install popular Linux distributions â€“ such as Ubuntu, Debian, and Fedora â€“ from the Microsoft Store.
  • Use Linux command-line tools (e.g. bash, ssh, git, grep, sed).
  • Access your Windows files from Linux and vice versa.
  • Combine Windows applications (like VS Code) with Linux tooling in one workflow.

WSL 1 vs WSL 2

There are two major WSL versions, each with its own architecture and use-cases:

WSL 1

  • Translates Linux system calls into Windows system calls.
  • Very fast file access when working mainly in the Windows filesystem (e.g. C:\).
  • Good compatibility for many CLI tools, but not full Linux kernel functionality.

WSL 2

  • Runs an actual Linux kernel in a lightweight virtual machine managed by Windows.
  • Much better compatibility with Linux applications, Docker, networking tools, etc.
  • Better performance for Linux-heavy workloads, especially when working within the Linux filesystem.

For most modern use-cases, especially development with Docker or more advanced tools, WSL 2 is recommended. You can still switch individual distributions between WSL 1 and WSL 2 depending on your needs.

Why use WSL?

WSL is especially useful if you:

  • Develop software that runs on Linux (web apps, backend services, scripts, etc.).
  • Need access to package managers like apt, dnf, or pacman.
  • Want to use Linux-only tools while staying in Windows.
  • Prefer working in a terminal-based environment without leaving Windows.

Combined with editors like Visual Studio Code or JetBrains IDEs, WSL gives you a powerful development setup where the code runs in Linux, but the UI runs on Windows.

High-level architecture

At a high level, WSL looks like this:

  1. Windows manages one or more Linux distributions installed from the Microsoft Store or manually.
  2. Each distribution has its own filesystem, configuration, and tools.
  3. Windows provides integration layers for:
    • Filesystem access (e.g. /mnt/c for the Windows C: drive).
    • Networking (WSL instances get IP addresses and can reach the internet and Windows).
    • Interop between Windows and Linux commands (e.g. launching notepad.exe from Linux).
