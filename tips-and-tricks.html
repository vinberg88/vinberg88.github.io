Tips & tricks for WSL

These tips help you optimize performance, keep your environment organized, and make WSL feel more integrated with Windows.

Performance: prefer the Linux filesystem

For heavy development tasks (builds, large repositories, containers), store your code in the Linux filesystem, for example: 

/home/<user>/projects/my-app

Accessing many small files directly on /mnt/c can be slower under WSL 2. Using the Linux filesystem gives better performance and fewer edge cases.

Use a custom default user

Some distributions let you set a default user instead of root. For example, with Ubuntu you can run in PowerShell: 

ubuntu config --default-user <username>

Use aliases and shell configuration

Just like on any Linux system, configure your shell to speed up common tasks. In ~/.bashrc or ~/.zshrc: 

alias ll='ls -la'
alias gs='git status'
alias ..='cd ..'
alias cproj='cd ~/projects'

Sharing SSH keys

Instead of keeping separate SSH keys in WSL, you can:

  • Store keys in WSL under ~/.ssh and configure Git to use them.
  • Or point WSL SSH to keys on your Windows drive (but be careful with permissions).

For Git, you can set your WSL-wide user configuration: 

git config --global user.name "Your Name"
git config --global user.email "you@example.com"

Environment variables and PATH

Windows adds some paths into WSL so you can run Windows executables. If you prefer a cleaner environment, you can adjust your PATH in your shell configuration: 

# Example: move Windows paths to the end (bash)
PATH="$PATH:/mnt/c/Windows:/mnt/c/Windows/System32"

Using Docker with WSL 2

Docker Desktop for Windows integrates with WSL 2. Recommended setup:

  1. Install Docker Desktop on Windows.
  2. Enable the WSL 2 backend in Docker settings.
  3. Select which WSL distributions Docker can access.

Then you can run Docker commands directly inside your WSL terminal: 

docker ps
docker run hello-world

Useful commands to remember

# List distributions
wsl --list --verbose

# Change default distribution
wsl --set-default <DistributionName>

# Change version (WSL 1 <-> WSL 2)
wsl --set-version <DistributionName> 2

# Export a distribution to a .tar file
wsl --export <DistributionName> C:\backups\ubuntu-backup.tar

# Import a distribution from a .tar file
wsl --import <NewName> C:\wsl\custom C:\backups\ubuntu-backup.tar --version 2
